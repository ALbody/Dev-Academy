🎯 ما هو Flutter؟

Flutter هو إطار عمل (Framework) مفتوح المصدر طورته Google في عام 2015، مصمم لتطوير تطبيقات محمولة عالية الأداء وعابرة للمنصات. يوفر Flutter تجربة تطوير موحدة للعمل مع نظامي iOS و Android من خلال قاعدة برمجية واحدة.

💡 المميزات الرئيسية:

  • Cross-Platform Development: كود واحد يعمل على iOS و Android و Web وحتى Desktop
  • Performance عالية: Flutter يستخدم محرك Skia لرسم الواجهات مباشرة، مما يوفر أداءً ممتازاً بمعدل 60-120 FPS
  • Hot Reload: تحديثات فورية للكود أثناء التطوير دون الحاجة لإعادة تشغيل التطبيق
  • Rich UI Widgets: مكتبة ضخمة من الواجهات الجاهزة والقابلة للتخصيص
  • Material و Cupertino Design: دعم كامل لمعايير التصميم الحديثة

🔧 البيئة التقنية:

لغة البرمجة: Dart - لغة حديثة موجهة للكائنات، سهلة التعلم وقوية جداً

المحرك: Flutter Engine يتم تجميعه كـ native code

الأدوات: Android Studio أو VS Code + Flutter SDK

2. استكشاف Widgets بعمق

Stage: Content

📚 أهداف التعلم:

  • فهم فلسفة Widget في Flutter
  • التمييز بين StatelessWidget و StatefulWidget
  • بناء Widget tree معقد وفعال
  • فهم دورة حياة الـ Widgets

🎯 ما هي Widgets؟

في Flutter، "كل شيء هو Widget" - هذا مبدأ أساسي يعني أن كل عنصر في الواجهة، من الأزرار والنصوص إلى التخطيطات والشاشات، يتم تمثيله كـ Widget. Widgets هي وحدات بناء صغيرة وقابلة لإعادة الاستخدام تصف جزءاً من الواجهة.

🔑 أنواع Widgets الرئيسية:

1. StatelessWidget

هو Widget ثابت لا يتغير حالته بعد الإنشاء. يُستخدم للعناصر التي لا تحتاج لتخزين معلومات متغيرة.

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('نص ثابت لن يتغير');
  }
}

2. StatefulWidget

هو Widget يمكنه تخزين حالة (State) ويمكن أن يعدل هذه الحالة بمرور الوقت، مما يؤدي لإعادة رسم الواجهة.

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int counter = 0;

  @override
  Widget build(BuildContext context) {
    return GestureDetector(
      onTap: () {
        setState(() {
          counter++;
        });
      },
      child: Text('تم الضغط $counter مرات'),
    );
  }
}

🌳 Widget Tree و Composition

تنظيم الـ Widgets في شكل شجرة هرمية (Tree) حيث كل Widget يمكن أن يحتوي على عدة Widgets فرعية، مما ينشئ بنية متسلسلة قوية.

📦 أمثلة على Widgets المدمجة:

  • Container: عنصر يوفر تحكماً على الحجم واللون والزوايا المستديرة والظل
  • Row/Column: لترتيب الـ Widgets أفقياً أو عمودياً
  • Text: عنصر النص مع خيارات تنسيق متقدمة
  • Button: أزرار قابلة للضغط برسوميات جميلة
  • Image: عرض الصور من مصادر مختلفة
  • TextField: حقول إدخال النصوص
  • ListView: قوائم قابلة للتمرير

🔄 دورة حياة StatefulWidget:

  1. يتم استدعاء createState()
  2. يتم استدعاء initState() - للتهيئة
  3. يتم استدعاء build() - رسم الواجهة
  4. عند تغيير الحالة، يتم استدعاء setState() و build() مجدداً
  5. يتم استدعاء dispose() - عند حذف الـ Widget

💡 أفضل الممارسات:

  • استخدم StatelessWidget عندما لا تحتاج لتخزين حالة
  • قسّم الواجهات الكبيرة إلى Widgets صغيرة قابلة لإعادة الاستخدام
  • تجنب البناء المعقد للـ Widget tree - استخدم extraction
  • استخدم const Constructors للـ Widgets الثابتة لتحسين الأداء

3. نظام التخطيطات (Layouts System)

Stage: Content

📚 أهداف التعلم:

  • فهم نظام الـ Layout في Flutter
  • إتقان استخدام Row و Column و Stack
  • فهم المحاذاة والمساحات
  • بناء تخطيطات معقدة وفعالة

🎯 أساسيات الـ Layout:

في Flutter، التخطيط (Layout) هو عملية ترتيب Widgets على الشاشة. يتم التحكم فيه من خلال Widgets خاصة تُسمى Layout Widgets التي تحدد كيفية ترتيب Widgets الفرعية.

📏 Row - الترتيب الأفقي

يرتب Widgets في صف واحد أفقياً (من اليسار إلى اليمين):

Row(
  mainAxisAlignment: MainAxisAlignment.spaceAround,
  crossAxisAlignment: CrossAxisAlignment.center,
  children: [
    Icon(Icons.home),
    Icon(Icons.search),
    Icon(Icons.settings),
  ],
)

📏 Column - الترتيب العمودي

يرتب Widgets في عمود واحد عمودياً (من الأعلى إلى الأسفل):

Column(
  mainAxisAlignment: MainAxisAlignment.start,
  crossAxisAlignment: CrossAxisAlignment.stretch,
  children: [
    Text('عنوان'),
    Text('وصف'),
    ElevatedButton(onPressed: (){}, child: Text('اضغط هنا')),
  ],
)

🎨 Stack - التراكب

يرتب Widgets فوق بعضها البعض، حيث الـ Widget الأول يكون في الأسفل والأخير في الأعلى:

Stack(
  children: [
    Container(color: Colors.blue, width: 200, height: 200),
    Positioned(
      top: 10,
      left: 10,
      child: Text('نص فوق الصورة'),
    ),
  ],
)

⚙️ خيارات المحاذاة:

  • MainAxisAlignment: تحكم في محاذاة Widgets على المحور الرئيسي
    • start, end, center, spaceBetween, spaceAround, spaceEvenly
  • CrossAxisAlignment: تحكم في محاذاة Widgets على المحور الفرعي
    • start, end, center, stretch, baseline

📐 Flex و Expanded

Flex: يوزع المساحة المتاحة بين الـ Widgets الفرعية

Expanded: يجعل Widget يأخذ كل المساحة المتاحة

Row(
  children: [
    Expanded(flex: 1, child: Container(color: Colors.red)),
    Expanded(flex: 2, child: Container(color: Colors.blue)),
  ],
)

🎯 Padding و Margin:

  • Padding: مساحة داخلية (بين محتوى الـ Widget وحدوده)
  • SizedBox: لإضافة مساحات فارغة محددة

💡 أفضل الممارسات:

  • استخدم Column/Row بدلاً من Absolute Positioning عندما يكون ممكناً
  • احذر من الـ Layout Issues عند دمج Widgets بطريقة خاطئة
  • استخدم Constraints بحكمة لتجنب مشاكل الأداء

4. إدارة حالة التطبيق (State Management)

Stage: Content

📚 أهداف التعلم:

  • فهم مفهوم State Management في تطبيقات Flutter
  • إتقان setState وفهم دورتها
  • فهم الأنماط المتقدمة مثل Provider و BLoC
  • اختيار الحل الأنسب لكل حالة استخدام

🎯 ما هي إدارة الحالة؟

إدارة الحالة تعني كيفية تخزين ومشاركة البيانات المتغيرة بين Widgets المختلفة. عندما تتغير البيانات، يجب تحديث الواجهة الرسومية للعكس عن هذه التغييرات.

🔧 setState - الطريقة البسيطة:

class Counter extends StatefulWidget {
  @override
  _CounterState createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int count = 0;

  void increment() {
    setState(() {
      count++;  // تحديث الحالة وإعادة رسم الواجهة
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('العدد: $count'),
        ElevatedButton(
          onPressed: increment,
          child: Text('زيادة'),
        ),
      ],
    );
  }
}

📊 StatefulWidget - دورة الحياة:

  1. initState(): يُستدعى مرة واحدة عند إنشاء الـ Widget - استخدمه للتهيئة
  2. build(): يُستدعى عند إعادة الرسم - يجب أن يكون سريعاً
  3. setState(): عند تغيير الحالة - يؤدي لاستدعاء build()
  4. dispose(): يُستدعى عند حذف الـ Widget - استخدمه للتنظيف

🏗️ الأنماط المتقدمة:

1. Provider Pattern (الموصي به)

نمط حديث وسهل يسمح بمشاركة البيانات عبر الـ Widget Tree بدون الحاجة لتمرير البيانات يدوياً:

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ChangeNotifierProvider(
      create: (context) => CounterModel(),
      child: MaterialApp(
        home: Home(),
      ),
    );
  }
}

class Home extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final counter = Provider.of<CounterModel>(context);
    return Text('${counter.count}');
  }
}

2. BLoC Pattern (للتطبيقات الكبيرة)

نمط معقد لكن قوي جداً يفصل منطق العمل عن الواجهة الرسومية

3. Riverpod (الجديد والحديث)

تحسين على Provider يوفر type safety أفضل والمزيد من الميزات

💡 قاعدة ذهبية:

  • تطبيق صغير؟ استخدم setState
  • تطبيق متوسط؟ استخدم Provider
  • تطبيق كبير معقد؟ استخدم BLoC أو Riverpod

⚠️ أخطاء شائعة:

  • استدعاء setState من خارج الـ Widget
  • عدم استدعاء dispose() للموارد
  • عدم استخدام const Constructors

5. نظام التنقل (Navigation System)

Stage: Content

📚 أهداف التعلم:

  • فهم نظام التنقل في Flutter
  • إتقان Navigator و Routes
  • نقل البيانات بين الشاشات
  • التعامل مع Deep Linking

🎯 أساسيات التنقل:

النقل بين الشاشات (Navigation) هو عملية الانتقال من شاشة إلى أخرى. في Flutter، يتم التحكم به من خلال Widget يسمى Navigator الذي يدير stack من الشاشات.

🔀 Navigator و Routes:

يعمل Navigator مثل Stack - عندما تنتقل لشاشة جديدة (Push)، تُضاف إلى الـ Stack. عندما تضغط الزر "رجوع" (Pop)، تُزال من الـ Stack.

📍 التنقل البسيط:

// الذهاب إلى شاشة جديدة
Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => SecondScreen()),
);

// الرجوع من الشاشة
Navigator.pop(context);

🏷️ Named Routes (الطريقة الموصى بها):

// تعريف المسارات في MaterialApp
MaterialApp(
  routes: {
    '/': (context) => Home(),
    '/details': (context) => Details(),
    '/settings': (context) => Settings(),
  },
)

// الانتقال باستخدام اسم المسار
Navigator.pushNamed(context, '/details');

// الرجوع
Navigator.pop(context);

📤 نقل البيانات بين الشاشات:

// إرسال البيانات
class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: () async {
        final result = await Navigator.push(
          context,
          MaterialPageRoute(
            builder: (context) => SecondScreen(data: 'Hello'),
          ),
        );
        print('النتيجة: $result');
      },
      child: Text('اذهب'),
    );
  }
}

// استقبال البيانات
class SecondScreen extends StatelessWidget {
  final String data;
  SecondScreen({required this.data});

  @override
  Widget build(BuildContext context) {
    return Text(data);
  }
}

🔄 الانتقالات مع الرسوميات:

Flutter يوفر عدة أنواع من الرسوميات الانتقالية (Page Transitions):

  • MaterialPageRoute: انتقال مثل Material Design
  • CupertinoPageRoute: انتقال نمط iOS
  • PageRouteBuilder: انتقال مخصص بالكامل

⚠️ أفضل الممارسات:

  • استخدم Named Routes لتنظيم أفضل
  • تجنب الـ Deep nesting من الشاشات
  • استخدم async/await لانتظار النتائج من الشاشات الجديدة

6. النماذج والتحقق من الصحة (Forms & Validation)

Stage: Content

📚 أهداف التعلم:

  • بناء نماذج احترافية في Flutter
  • التحقق من صحة البيانات المُدخلة
  • التعامل مع أنواع لوحات المفاتيح المختلفة
  • تحسين تجربة المستخدم في إدخال البيانات

🎯 مكونات النموذج:

النموذج (Form) هو مجموعة من حقول الإدخال التي تجمع البيانات من المستخدم. في Flutter، يتم استخدام Form widget مع TextFormField للتحقق من الصحة.

📝 بناء نموذج بسيط:

class MyForm extends StatefulWidget {
  @override
  _MyFormState createState() => _MyFormState();
}

class _MyFormState extends State<MyForm> {
  final _formKey = GlobalKey<FormState>();
  String email = '';
  String password = '';

  @override
  Widget build(BuildContext context) {
    return Form(
      key: _formKey,
      child: Column(
        children: [
          TextFormField(
            decoration: InputDecoration(
              labelText: 'البريد الإلكتروني',
              border: OutlineInputBorder(),
            ),
            validator: (value) {
              if (value?.isEmpty ?? true) {
                return 'الرجاء إدخال البريد';
              }
              if (!value!.contains('@')) {
                return 'البريد غير صحيح';
              }
              return null;
            },
            onSaved: (value) => email = value ?? '',
          ),
          SizedBox(height: 16),
          ElevatedButton(
            onPressed: () {
              if (_formKey.currentState!.validate()) {
                _formKey.currentState!.save();
                print('البريد: $email');
              }
            },
            child: Text('إرسال'),
          ),
        ],
      ),
    );
  }
}

🔍 التحقق من الصحة (Validation):

التحقق من الصحة يتم من خلال دالة validator التي تُرجع رسالة خطأ أو null إذا كانت البيانات صحيحة.

  • Email Validation: التحقق من صيغة البريد الإلكتروني
  • Length Validation: التحقق من طول النص
  • Pattern Validation: التحقق من نمط معين (مثل أرقام فقط)
  • Custom Validation: تحقق مخصص

⌨️ أنواع لوحات المفاتيح:

TextFormField(
  keyboardType: TextInputType.email,  // لوحة مفاتيح البريد
)

TextFormField(
  keyboardType: TextInputType.number,  // لوحة مفاتيح الأرقام
)

TextFormField(
  keyboardType: TextInputType.phone,  // لوحة مفاتيح الهاتف
)

🎨 تحسين مظهر الحقول:

TextFormField(
  decoration: InputDecoration(
    labelText: 'اسمك',
    hintText: 'أدخل اسمك الكامل',
    border: OutlineInputBorder(),
    prefixIcon: Icon(Icons.person),
    suffixIcon: Icon(Icons.check),
  ),
)

💡 أفضل الممارسات:

  • استخدم Form و GlobalKey للتحقق من جميع الحقول دفعة واحدة

🎯 أساسيات API Integration:

📦 مكتبة HTTP:

import 'package:http/http.dart' as http;

// طلب GET بسيط
Future<void> fetchData() async {
  try {
    final response = await http.get(
      Uri.parse('https://api.example.com/posts'),
    );
    
    if (response.statusCode == 200) {
      // نجاح الطلب
      print('البيانات: ${response.body}');
    } else {
      // فشل الطلب
      print('خطأ: ${response.statusCode}');
    }
  } catch (e) {
    print('خطأ الاتصال: $e');
  }
}

📤 أنواع الطلبات:

  • POST: إرسال البيانات إلى الخادم

  • DELETE: حذف البيانات من الخادم

📝 طلب POST مثال:

Future<void> createPost(String title, String body) async {

      'title': title,
      'body': body,
    }),
  );
  
  if (response.statusCode == 201) {
    print('تم الإنشاء بنجاح');
  }
}

val length = email?.length  // إذا كان email null، سيكون الناتج null
val length2 = email?.length ?: 0  // إذا كان null، سيكون 0

val length3 = email!!.length  // سيرفع exception إذا كان null

    print('يوم عمل');
    break;
  default:
    print('يوم غير معروف');
}

🎯 البرمجة الكائنية - مقدمة:

البرمجة الكائنية (OOP) هي طريقة في البرمجة تعتمد على تمثيل العالم الحقيقي من خلال Objects (كائنات) التي تحتوي على البيانات والسلوكيات.

🔧 Classes و Objects:

// تعريف فئة (Class)
class Dog {
  String name;
  int age;
  
  // Constructor
  Dog(this.name, this.age);
  
  // Method
  void bark() {
    print('$name يعوي: وووف!');
  }
  
  void info() {
    print('$name عمره $age سنوات');
  }
}

// إنشاء object (كائن)
void main() {
  Dog dog1 = Dog('ماكس', 3);
  dog1.bark();
  dog1.info();
}

🔗 الوراثة (Inheritance):

تسمح للفئة الجديدة بوراثة الخصائص والدوال من فئة قديمة.

// الفئة الأساس (Parent Class)
class Animal {
  String name;
  
  Animal(this.name);
  
  void eat() {
    print('$name يأكل');
  }
}

// فئة مشتقة (Child Class)
class Cat extends Animal {
  String color;
  
  Cat(String name, this.color) : super(name);
  
  void meow() {
    print('$name يموء: ميووو');
  }
  
  @override
  void eat() {
    print('$name يأكل السمك');
  }
}

void main() {
  Cat cat = Cat('سوسو', 'أبيض');
  cat.eat();     // سوسو يأكل السمك
  cat.meow();    // سوسو يموء: ميووو
}

📋 Abstract Classes:

abstract class Shape {
  double area();
  
  void describe() {
    print('هذا شكل بمساحة ${area()}');
  }
}

class Circle extends Shape {
  double radius;
  
  Circle(this.radius);
  
  @override
  double area() => 3.14 * radius * radius;
}

void main() {
  Shape circle = Circle(5);
  circle.describe();  // هذا شكل بمساحة 78.5
}

🔐 Getters و Setters:

class BankAccount {

18. البرمجة غير المتزامنة (Asynchronous Programming)

Stage: Content

📚 أهداف التعلم:

  • استخدام Futures و async/await
  • التعامل مع Streams
  • معالجة الأخطاء في العمليات غير المتزامنة

🎯 لماذا البرمجة غير المتزامنة؟

في بعض الأحيان، يستغرق تنفيذ عملية معينة وقتاً طويلاً (مثل تحميل البيانات من الإنترنت). البرمجة غير المتزامنة تسمح للبرنامج بمتابعة العمل بينما ننتظر نتيجة العملية.

⏳ Future - العملية المستقبلية:

Future يمثل قيمة ستتوفر في المستقبل (أو خطأ).

Future<String> fetchUserName() {
  return Future.delayed(Duration(seconds: 2), () {
    return 'أحمد';
  });
}

void main() {
  print('البداية');
  
  fetchUserName().then((name) {
    print('اسم المستخدم: $name');
  }).catchError((error) {
    print('خطأ: $error');
  }).whenComplete(() {
    print('انتهت العملية');
  });
  
  print('النهاية');
}

// الإخراج:
// البداية
// النهاية
// اسم المستخدم: أحمد
// انتهت العملية

⚡ async/await - الطريقة الحديثة والأسهل:

🔀 انتظار عمليات متعددة:

Future<String> fetchUser() => 
  Future.delayed(Duration(seconds: 1), () => 'أحمد');

Future<String> fetchPosts() => 
  Future.delayed(Duration(seconds: 2), () => '10 منشورات');

// انتظار الاثنين معاً
void main() async {
  try {
    final results = await Future.wait([
      fetchUser(),
      fetchPosts(),
    ]);
    
    print('المستخدم: ${results[0]}');
    print('المنشورات: ${results[1]}');
  } catch (e) {
    print('خطأ: $e');
  }
}

💧 Streams - تدفق البيانات:

Stream يشبه Future لكنه يرسل عدة قيم بمرور الوقت، وليس قيمة واحدة فقط.

// إنشاء Stream
Stream<int> countStream() async* {
  for (int i = 1; i <= 5; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;  // إرسال قيمة
  }
}

// الاستماع إلى Stream
void main() async {
  await for (int count in countStream()) {
    print('العدد: $count');
  }
  print('انتهى العد');
}

📡 StreamController - التحكم في Streams:

void main() {
  final controller = StreamController<String>();
  
  // الاستماع
  controller.stream.listen((value) {
    print('تم استقبال: $value');
  });
  
  // إرسال بيانات
  controller.add('السلام عليكم');
  controller.add('ورحمة الله');
  controller.add('وبركاته');
  
  controller.close();
}

⚠️ معالجة الأخطاء:

Future<String> riskOperation() async {

void main() async {

    String result = await riskOperation();

  } on Exception catch (e) {

💡 أفضل الممارسات:

  • استخدم async/await بدلاً من then/catchError
  • تعامل مع الأخطاء بشكل محدد باستخدام on
  • استخدم Future.wait عند الحاجة لانتظار عمليات متعددة
  • استخدم Streams للبيانات التي تتغير بمرور الوقت

🔗 مثال عملي - تحميل البيانات:

class DataRepository {
  Future<List<String>> fetchData() async {
    // محاكاة تأخير الشبكة

    // محاكاة البيانات
    return ['العنصر 1', 'العنصر 2', 'العنصر 3'];
  }
}

void main() async {
  final repo = DataRepository();
  
  print('جاري التحميل...');
  try {
    final data = await repo.fetchData();
    print('تم التحميل بنجاح:');
    for (var item in data) {
      print('  - $item');
    }
  } catch (e) {
    print('فشل التحميل: $e');
  }
}
